# Описания методов получения данных из БД

## Переключение на окно с чатами (метод switch_to_chatting_page в классе [App](../../UI/main.py), выполняется после регистрации/входа)

При первом переключении получаем данные чатов пользователя для последующего вывода на экран.

### Запросы на получения данных:
* Получение всех чатов пользователя
  - **Принимает**: id пользователя
  - **Возвращает**: словари с id чата, его типом и именем
    
* Получение всех пользователей чата
  - **Принимает**: id чата
  - **Возвращает**: словари с id пользователя, именем пользователя для каждого пользователя
    
* Получение последнего сообщения и имени его отправителя
  - **Принимает**: id чата
  - **Возвращает**: строки с последним сообщением и именем его отправителя, если в чате нет сообщений то возвращает 'Начните общение!'
    
## Открытие чата (метод open_chat в классе [App](../../UI/main.py), выполняется после нажатия на selectable_chat_view)

Принимает id чата, который нужно открыть

### Запросы на получение данных:
* Получение данных сообщений в чате
  - **Принимает**: id чата
  - **Возвращает**: user_id, file_id, type, text сообщения в виде словаря
    
* Получение имени отправителя для каждого сообщения
  - **Принимает**: user_id сообщения
  - **Возвращает**: единственную строку с именем пользователя
    
## Регистрация (метод register_user в классе [App](../../UI/main.py), выполняется после нажатия кнопки )

Отвечает за регистрацию пользователя в мессенджере

### Запросы на получение данных:
* Проверка на существование пользователя с таким именем пользователя
  - **Принимает**: username пользователя
  - **Возвращает**: существует ли пользователь с таким именем пользователя: bool

* Регистрация пользователя
  - **Принимает**: name, username, password пользователя
  - **Возвращает**: Всю информацию о созданном пользователе из таблицы [Users](../../Database/models/users.py) (для возможности её отображения и добавления пользователя в список участников чата)
