# Описание необходимых изменений/добавлений на сервер
ВАЖНО: в возврате написана минимальная нужная инфа на данный момент, но её может стать больше, и если БД возвращает больше, то лучше вернуть всю информацию
- Уникальное имя пользователя - username: str
- Отображаемое имя пользователя - name: str
- Уникальный номер - id: int
- Тип - type: str
- Имя (name) отправителя - sender_name: str
- Ссылка на аватар - avatar_url: str
- Пользователь-участник - участник чата, информация о котором, тем не менее, берётся из таблицы [users](../../Database/models/users.py)
- Список пользователей-участников чата - participants: list(dict)
## Изменения
* Заменить метод open_chat на метод get_messages, который:
  
  - **Принимает**: id чата
  - **Возвращает**: список из type, text или fail_id (в зависимости от типа), sender_name в виде словаря для установленного количества последних сообщений

* Добавить в методs auth возвращение всех данных пользователя:

  - **Принимает**: username входящего в auth
  - **Возвращает**: все данные о пользователе из таблицы [users](../../Database/models/users.py)
 
* Добавить возвращение всех данных пользователя после создания в метод registration:

  - **Принимает**: name, username, password создаваемого пользователя
  - **Возвращает**: все данные о пользователе из таблицы [users](../../Database/models/users.py)
 
## Добавления
* Добавить метод обработки данных чата get_chat_data, который:

  - **Принимает**: id чата
  - **Возвращает**: type, text или file_id, sender_name последнего сообщения + participants

* Добавить метод создания чата create_chat, который:

  - **Принимает**: type, name, avatar_url, participants создаваемого чата
  - **Возвращает**: id нового чата
