# Вспомогательные операции
  ## Регистрация (класс App в main.py) register_user()
  * Метод класса приложения, обрабатывает нажатие кнопки подтверждения регистрации на sign_up_page
  * Производимые действия
    - Считать данные для регистрации из name_entry, username_entry, password_entry и password_confirm_entry которые находятся sign_up_page
    - Проверить заполненность полей и совпадение пароль и его подтверждения
    - Проверить существование пользователя по имени пользователя
    - Если проверка прошла успешно, отправить запрос на регистрацию, установить значения переменной user в классе User на данные пользователя и is_authorized на True и перейти на chatting_page
    - Если проверка не удалась, показать пользователю сообщение об ошибке
  * Взаимодействия с БД
    - Проверка на существование (на вход: username: str, на выход: bool)
    - Запрос на регистрирование (на вход: name: str, username: str, password: str, на выход: данные пользователя(если успех): dict, иначе вернуть текст ошибки: str)
  ## Авторизация (класс App в main.py) login_user()
  * Метод класса приложения, обрабатывает нажатие кнопки подтверждения авторизации на sign_up_page
  * Проиводимые действия
    - Считать данные из password_entry и username_entry, который находится в sign_up_page
    - Проверить заполненность полей
    - Проверить существование пользователя с такими username и password
    - Если проверка прошла успешно, записать данные пользователя в переменную user, получив их по имени пользователя
    - Если проверка не удалась, показать пользователю сообщение об ошибке
  * Взаимодействие с БД
    - Проверка существования (на вход: username: str, password: str, на выход: bool)
    - Запрос на вход (на вход: username: str, на выход: записать данные пользователя в переменную user (если успешно), иначе вывести сообщение об ошибке)
  ## Получить название текущей страницы (класс App в main.py) get_current_page()
  * Возвращает строку с названием текущего окна (страницы) в соответствии с названием файла для этой страницы
  * Производимые действия
    - Вернуть значение переменной name класса текущей страницы
  ## Открытие чата (класс App в main.py) open_chat(chat_id)
  * Метод класса приложения, обрабатывает нажатие на чат ЛКМ для его выбора
  * Производимые действия
    - Получить данные из созданного ранее массива App.available_chats по chat_id
    - Для каждого сообщения из чата определить имя (name) его отправителя
    - Добавить вид чата на экран пользователя
  * Взаимодействие с БД
    - Определить имя отправителя (на вход: параметр user_id сообщения: str, на выход: параметр name пользователя: str)
