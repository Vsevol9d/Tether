# Database/[models](../../Database/models)/ - классы с шаблонами таблиц

#### Примечание:  
В каждой таблице (иначе - модели) есть **названия колонок** и **строчки**.  
**Строчки** - инфа, которая хранится в БД (например: 0, Разработка немыслимого, 02-05-2026, group)  
**Названия колонок** - всегда одинаковые для каждой строки в таблице ключ-слова  
У каждой колонки есть свои *ограничения* (например: только str, максимальная длина значения в колонке - 60 символов, не может быть пустой)  
Ниже они будут приведены в следующем формате:  
<название колонки>: <тип данных>(<максимальное количество символов для типа str>), <дополнительные ограничения>  
Дополнительные ограничения:
  * **PK** - первичный ключ (primary key) должен быть уникальным в колонке, не пустым, задаётся автоматически. По нему осуществляется удаление и обновление строк. Если в одной модели 2 и более первичных ключей, то при удалении или обновлении строки нужно указывать их все  
  * **unique** - значения в колонке должны быть уникальны, иначе ошибка
  * **NotNull** - поле обязательно к заполнению
  * **default=<значение>** - значение, которое будет подставлено, если ничего не указать
  * **allowed(<значения>)** - набор значений, которые может принимать колонка, иначе ошибка  
  * **format=<значение>** - для дат и времён, если не указано
  
Например, date_created: str(10), NotNull, default(текущее время), format=DD-MM-YYYY

## класс [Users](../../Database/models/users.py) (таблица users):
#### Инфо обо всех пользователях
  * **id**: int, PK
  * **name**: str(52), NotNull
  * **username**: str(30), NotNull, unique
  * **lastname**: str(60)
  * **birthday**: str(10), format=DD-MM-YYYY
  * **avatar_url**: int, имя файла в assets/avatar_users/...
  * **date_created**: str(10), NotNull, default=текущая дата, format=DD-MM-YYYY
  * **last_time_online**: str(19), NotNull, default=текущая дата и время, format=DD-MM-YYYY HH:MI:SS 
  * **phone**: str(20), unique, format=+7 9xx xxx xx xx
  * **email**: str(254), unique
   
## класс [Chats](../../Database/models/chats.py) (таблица chats):
#### Инфо обо всех чатов (схоже по смыслу с таблицей users)
  * **id**: int, PK
  * **name**: str(60), NotNull
  * **avatar_url**: int, имя файла в assets/avatar_chats/...
  * **date_created**: str(10), default=текущая дата, format=DD-MM-YYYY
  * **type**: str, NotNull, default="private", allowed("private", "group", "channel")
  
## класс [Messages](../../Database/models/messages.py) (таблица messages):
#### Инфо обо всех сообщениях
  * **id**: int, PK
  * **type**: str, NotNull, default="text", allowed("text", "file", "image", "video")
  * **text**: str(4096), если >4096, то текст разделяется несколько сообщений
  * **file_id**: int, имя файла в assets/media/...
  * **creation_date_time**: str(19), NotNull, defualt=текущая дата и время, format=DD-MM-YYYY HH:MI:SS
  * **chat_id**: int, PK
  * **user_id**: int, NotNull
    
## класс [Participants](../../Database/models/participants.py) (таблица participants):
#### Инфо обо всех пользователях как "участниках чатов"
  * **chat_id**: int, PK
  * **user_id**: int, PK
  * **joined_at**: str(19), default=текущая дата и время, format=DD-MM-YYYY HH:MI:SS
  * **role**: str: NotNull, default="Участник", allowed("Участник", "Пре-админ", "Админ")

## класс [Friend](../../Database/models/friends.py) (таблица friends):
#### Инфо обо всех связях между пользователями
  * **user_id_1**: int, PK
  * **user_id_2**: int, PK
  * **level_relationships**: int, NotNull, default=1, allowed(1, 2, 3, 4)
    - 1 - знакомые - 2 пользователя состоят в одной группе
    - 2 - приятели - имеется личная переписка между двумя пользователями
    - 3 - друзья - знают телефон друг друга
    - 4 - лучшие друзья - пользователь сам их помечает (на будущие)
