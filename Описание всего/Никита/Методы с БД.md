## isSuccess: все api-функции БД возвращают isSuccess-словарь с одинаковой структурой
### Если при выполнении функции не возникло ошибки:  
```json lines
{
  'isSuccess': True, 
  'data': <...>
}
``` 
data может принимать типы *list*, *dict* и *bool* в разных функциях.  
Если написано *isSuccess без data*, это значит, что при успехе операции вернётся *{'isSuccess': True}*

### Если при выполнении функции возникла ошибка:
```json lines
{  
  'isSuccess': False,  
  'error': traceback.format_exc(),
  'long_error': traceback.format_exc()  
}
```
error - краткое сообщение об ошибке, например ```ZeroDivisionError: division by zero```.
long_error - сообщение об ошибке с полным *Traceback*


## Есть 5 [базовых методов](../../Database/methods/basic_methods.py), которые работают со всеми таблицами:
Чтобы их вызвать, нужно написать *db.<таблица>.<метод>* (например, db.users.select_all())
  * add(...)
  * update(...)
  * delete(...)
  * select_all()
  * exists(...)


### Функция add(...) -> dict
*Добавляет новую строку с данными в соответствующую таблицу.* 

За **параметры** функции брать названия колонок в таблице.  
За **аргументы** - значения, нуждающийся во вставке в эту колонку.  
*Исключениями* являются колонки "id", "date_created", "creation_date_time", "joined_at".  

**Возвращает** словарь isSuccess, где data содержит стандартный словарь, но без колонок-исключений и все значения, 
указанные как *None*, но имеющие значение по умолчанию, вернут *None*


### Функция update(id: int | list, attr_name: str, value: Any) -> dict
*Обновляет строчку в таблице, через её id.*

**id** - первичный ключ строки. Если ПК у строки 2 и более, то брать их все через список.
**attr_name** - название колонки, значение которой нужно обновить.
**value** - значение, на которое нужно обновить поле.

**Возвращает** isSuccess без data ({'isSuccess': True}).


### Функция delete(id: int | list) -> dict
*Удаляет строчку в таблице, через её id.*

**id** - первичный ключ строки. Если ПК у строки 2 и более, то брать их все через список.

**Возвращает** isSuccess без data.


### Функция select_all() -> dict
*Получает все строки в таблице.*

*Параметров нет.*

**Возвращает isSuccess, где data содержит список строк, представленных в виде словарей.**

### Функция exists(...) -> dict
**Проверка существования значения в определённой строке таблицы**

За **параметры** функции брать названия колонок в таблице.  
За **аргументы** - значения, нуждающийся в проверке наличия в данной колонке.  
**Например:** *exists(username='Nikita', name='Nik')*

**Возвращает** isSuccess, гда data содержит:
   * *True*, если значение существует.
   * *False*, если значение не существует, или неверное имя колонки.

**Например:** {'isSuccess': True, 'data': False}
